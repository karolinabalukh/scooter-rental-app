#  Postman Flow: Тестування повного циклу оренди самоката

Цей сценарій описує покрокову перевірку роботи мікросервісної системи оренди самокатів.  
Запити виконуються в строгій послідовності для коректного проходження бізнес‑логіки.

---

## Крок 1. Реєстрація та авторизація (Auth Service – порт `8081`)
### 1.1 Реєстрація нового користувача
- **Метод:** `POST`
- **URL:** `http://localhost:8081/api/auth/register`
- **Тіло запиту (JSON):**
  ```json
  {
    "email": "email@gmail.com",
    "password": "000"
  }

---
## Крок 2. Підготовка транспорту (Scooter Service - 8082)
### Створення самоката:
- **Метод:** POST
- **URL:** http://localhost:8082/api/scooters
- **Тіло запиту (JSON):**
```json
{ "serialNumber": "BOLT-001", "battery": 100, "latitude": 49.8, "longitude": 24.0 } 
```


### Логін (Отримання токена):
- **Метод:** POST
- **URL:** http://localhost:8081/api/auth/login
- **Дія :** Скопіюйте довгий рядок токена з відповіді.
 

--- 

## Крок 3: Оренда та Поїздка (Ride Service - 8083)

у Postman: Authorization -> Type: Bearer Token -> вставте скопійований токен.

## Початок поїздки:
- **Метод:** POST
- **URL:** http://localhost:8083/api/rides/start?userId=1&scooterId=1
- **Результат:**  Статус самоката в БД scooter_db автоматично зміниться на IN_USE.

## Завершення поїздки:
- **Метод:** POST
- **URL:** http://localhost:8083/api/rides/3/end (замініть 3 на ID вашої поїздки).
- **Результат:** Ви отримаєте об'єкт поїздки з розрахованою ціною (price), а самокат знову стане AVAILABLE.
---
